# REVISÃO

---

> 103.1

-> type != witch

-> .bash_history (precisa incluir o ponto)

-> env declare set 

> 103.2

-> tr (revisar comandos como -s)

-> gerar uma saída cortando uma parte do texto (cut)

-> cr / lr

-> uso do sed para remover uma linha ou remover uma parte de um texto

-> sed por padrão não altera um arquivo, 
- para alterar de fato precisa usar a opção (-i) para editar o arquivo
- só altera a saída 

-> cat / tail / head / sort (-4) / split
- o padrão do split é reparar por arquivos de 1000 linhas

> 103.3

-> tar sempre precisar conter o (-f) para indicar o arquivo
- t para exibir
- x para extrair

-> ls -i (inodes)

-> find --name --user

-> criar uma imagem da partição (dd if=/dev/etc of=backup.img)

> 103.4

-> >filename 2>&1 ??????????????

-> myapp | tee -a (apende, escreve no final) log.out

-> xargs = construir e executar comandos a partir de uma entrada padrão

-> entender a dinâmica do pipe

> 103.5

-> SIGHUP (1) reinicializa e aplica as configurações mais recentes em um comando/arquivo

-> bg joga para background o último processo executado (o fg faz o contrário)

-> watch -n5 (executado a cada 5 segundos)

-> sinais sigup, sigint, sigterm, sigkill

-> nohup, fg e bg e &

-> uso no ps

-> PID (id do processo) e PPID (id do pai do PID)

-> load avarege (free)

-> pkill mata processos baseado em caracteristas (do grupo, do usuario, por nome)

-> screen vs. tmux

> 103.6

-> nice == define o niceness == o processo indiretamente 

-> PRIORITY != NICENESS

-> alterar o nice de um processo em execução é o renice ou top

-> -20 até o +18 

-> renice por grupo (-g) e usuário (-u)

> 103.7

-> símbolos das expressões regulares
- + encontra o caractere anterior uma ou mais vezes
- -> *imagem pdf*

-> egrep verbose (-v) mostra só as linhas onde as string não foram encontradas
- -h
- -l

-> exercícios de regex ebook (learning LPI)
