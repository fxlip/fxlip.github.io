opções de navegação do less

- `/` procura uma palavra e vai até ela
- `n` mais resultados para frente
- `N` mais resultados para trás
- `[CTRL+G]` status do arquivo
- `arquivolongo.txt lines 1-20/93 byte 1077/4965 22%  (press RETURN)`
- `q` para sair

observação sobre o pipe >>

- `|` vai usar a **saída do primeiro** comando como **entrada do segundo**

na prática o segundo comando controla a saída do primeiro

fxlip@www:~/files/lpi1$ sort --help
Uso: sort [OPÇÃO]... [ARQUIVO]...
 ou: sort [OPÇÃO]... --files0-from=A
Escreve de forma ordenada a concatenação do(s) ARQUIVO(s) na saída padrão.

Se ARQUIVO não for especificado ou for -, lê a entrada padrão.

Argumentos obrigatórios para opções longas também o são para opções curtas.
Opções de ordenação:

  -b, --ignore-leading-blanks  ignora espaços-brancos iniciais
  -d, --dictionary-order      considera apenas espaços-brancos e caracteres
                                alfanuméricos
  -f, --ignore-case           ignora diferença entre maiúsculas e minúsculas
  -g, --general-numeric-sort  compara de acordo com um valor numérico geral"
  -i, --ignore-nonprinting    considera apenas caracteres exibíveis"
  -M, --month-sort            compara (desconhecido) < "JAN" < ... < "DEZ"
  -h, --human-numeric-sort    compara números legíveis por humanos (ex., 2K 1G)
  -n, --numeric-sort          compara de acordo com o valor numérico do texto
  -R, --random-sort           embaralha agrupando chaves idênticas; veja shuf(1)
      --random-source=ARQUIVO obtém bytes de aleatoriedade do ARQUIVO
  -r, --reverse               inverte o resultado das comparações
      --sort=PALAVRA          ordena conforme PALAVRA:
                                general-numeric -g (numérico geral),
                                human-numeric -h (numérico humano),
                                month -M (mês), numeric -n (numérico),
                                random -R (aleatório), version -V (versão)
  -V, --version-sort          ordem natural de números (de versão) com texto

Outras opções:

      --batch-size=NMESCL     mescla no máximo NMESCL entradas de uma vez;
                                para maior uso de arquivos temporários
  -c, --check, --check=diagnose-first  verifica se está ordenada; não ordena
  -C, --check=quiet, --check=silent  como -c, mas não relata a primeira linha
                                       errada
      --compress-program=PROG  comprime temporários com PROG;
                               descomprima-os com PROG -d
      --debug               anota a parte da linha usada para ordenar,
                              e avisa sobre uso questionável para a saída
                              de erro padrão
      --files0-from=A       lê a entrada dos arquivos especificados por
                              nomes terminados em NULO no arquivo A;
                              se A for -, lê nomes da entrada padrão
  -k, --key=POS             ordena por um chave; POS fornece localização e tipo
  -m, --merge               mescla os arquivos já ordenados; não ordena
  -o, --output=ARQUIVO      escreve o resultado para ARQUIVO, e não para a
                              saída padrão
  -s, --stable              estabiliza a ordenação desabilitando a comparação
                              da última esperança ("last-resort")
  -S, --buffer-size=TAM     usa TAManho para o "buffer" de memória principal
  -t, --field-separator=SEP  usa SEP em vez de transições de espaços não
                              brancos para espaços-brancos
  -T, --temporary-directory=DIR  usa DIR para temporários, e não $TMPDIR ou
                              /tmp; múltiplas opções especificam múltiplos
                              diretórios
      --parallel=N          altera o número de ordenações simultâneas para N
  -u, --unique              com -c, verifica se a ordenação é estrita;
                              sem -c, emite apenas a primeira linha de uma
                              sequência repetida
  -z, --zero-terminated     termina linhas com byte 0, e não com nova linha
      --help        display this help and exit
      --version     output version information and exit

POS é F[.C][OPÇÕES][,F[.C][OPÇÕES]] para posições de início e parada, onde F é o número de um campo e C é a posição do caractere no campo; ambos são contados a partir de 1, e a posição de parada padrão é o final da linha. Se -t nem -b está ativa, os caracteres em um campo são contados a partir do início de um espaço branco precedente. OPÇÕES se compõem de uma ou mais opções (de uma letra) de ordenação [bdfgiMhnRrV], as quais substituem as opções globais de ordenação para aquela chave. Se nenhuma chave for fornecida, usa a linha inteira como chave. Use --debug para diagnosticar o uso incorreta da chave.

TAMANHO pode ser seguido pelos seguintes sufixos multiplicadores:
% 1% de memória, b 1, k 1024 (padrão), e assim por diante com M, G, T, P, E, Z e Y.

*** AVISO ***
A localidade especificada no ambiente afeta a ordem de classificação.
Defina LC_ALL=C para obter a ordenação tradicional que usa valores nativos de bytes.

 ou: od [-abcdfilosx]... [ARQUIVO] [[+]DESLOCAMENTO[.][b]]
 ou: od --traditional [OPÇÃO]... [ARQUIVO] [[+]DESLOCAMENTO[.][b] [+][RÓTULO][.][b]]

Escreve uma representação sem ambiguidade, em bytes octal por padrão, de ARQUIVO para a saída padrão. Com mais de um argumento ARQUIVO, concatena-os na ordem listada para formar a entrada.

Se ARQUIVO não for especificado ou for -, lê a entrada padrão.

Se tanto o primeiro como o segundo formato de chamada se aplicarem, o segundo é assumido se o último operando começar com + ou (se houverem 2 operandos) com um dígito.

Um operando DESLOCAMENTO significa o mesmo que -j DESLOCAMENTO. RÓTULO é um pseudo-endereço do primeiro byte emitido, incrementado à medida que progride o despejo de bytes.

Para DESLOCAMENTO e RÓTULO, um prefixo 0x ou 0X indica um numeral hexadecimal; sufixos possíveis são . para octal e b para multiplicação por 512.

  --help                      display this help and exit
  --version                   output version information and exit


As especificações tradicionais de formatação podem ser misturadas; elas
acumulam:
  -a    o mesmo que -t a,  seleciona caracteres com nome, ignorando o bit
        mais significativo
  -b    o mesmo que -t o1, seleciona bytes em octal
  -c    o mesmo que -t c,  seleciona caracteres exibíveis ou escapes por
        barra invertida
  -d    o mesmo que -t u2, seleciona decimais sem sinal de 2-bytes
  -f    o mesmo que -t fF, seleciona números reais (ponto flutuante)
  -i    o mesmo que -t dI, seleciona inteiros na base decimal
  -l    o mesmo que -t dL, seleciona inteiros longos na base decimal
  -o    o mesmo que -t o2, seleciona números em octal de 2 bytes
  -s    o mesmo que -t d2, seleciona números em decimal de 2 bytes
  -x    o mesmo que -t x2, seleciona números em hexadecimal de 2 bytes


TIPO é constituído de uma ou mais destas especificações:
  a          caractere com nome, ignorando o bit mais significativo
  c          caractere exibível ou escape por barra invertida
  d[TAM]     decimal com sinal (inteiro), TAM bytes por número
  f[TAM]     ponto flutuante (real), TAM bytes por float
  o[TAM]     octal, TAM bytes por número
  u[TAM]     decimal sem sinal (natural), TAM bytes por número
  x[TAM]     hexadecimal, TAM bytes por número

TAM é um número. Se TIPO é um dos [doux], TAM pode ser C para sizeof(char), S para sizeof(short), I para sizeof(int), ou L para sizeof(long). - `sed` é uma ferramenta muito poderosa, possui muitas funcionalidades e formas de uso, usado principalmente em conjunto com expressões regulares

- `sed 's/.*'` usado para procurar e substituir a primeira ocorrência de uma string
- `sed 's/.*/g'`  usado para procurar e substituir todas ocorrências de uma linha
- `sed '3,5 d'`  usado para apagar um range de linhas
- `sed '/Claudia/d'`  apaga a linha que contem uma string
