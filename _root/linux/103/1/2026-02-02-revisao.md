> **103.1**

---


Declaração de variáveis é case sensitive.

Uso das aspas não protege: `$`, `/` e ` mas \ protege.

Variáveis de ambiente: TUDO MAIÚSCULO ex: `PATH`

Variáveis de ambiente não existem em outros bashs (apenas local)

-> export para existir para outros bashs (apenas filhas, não se aplica a novas janelas/abas)

Mais em: @linux/105/2/

---

Outras formas de declarar: 
- echo ${VAR1}
- echo ${#VAR1} (exibe quantos caracteres)


`set` & `declare`

`set` comando externo, para controlar um ambiente, faz parte do bash.
- `set -a` marca variáveis, que foram modificadas ou criadas, para exportação.
- `set -x` mostra comandos e seus argumentos na medida em que forem executados.

`declare` declara variaveis -x
- `declare -x` pra fazer NOMEs exportar

`env -u` desdeclara uma variavel

`unset` desdeclara uma variavel

`$` para referenciar uma variável

`history`  auxiliar do usuário, não serve para auditoria
- pega tudo que está na memória e grava depois de fechar o terminal
- `!NNNN` para executar um comando pelo código
- `history -c` para limpar

`type` para mostrar onde está o comando

`uname -a` mostra todas as informações do SO

- `versão do kernel` = qunando o kernel foi COMPILADO
- `release do kernel` = versão do kernel `uname -a`


`whereis` pega nos principais diretórios os comandos 

---

`which` pega só oq está no PATH e mostra onde está o executável de um comando

-> retorna os nomes de caminho dos ficheiros (ou links) que podem ser executados no ambiente actual, tendo os seus argumentos sendo dados como comandos  numa  shell estritamente  compatível  com POSIX. Ele faz isto ao pesquisar a PATH por ficheiros executáveis que correspondem aos nomes dos argumentos. Não  canoniza  os  nomes  de caminho.

---

Qual comando roda em um ambiênte modificado? `env`

`man man`

-> é  o  paginador de manual do sistema. Cada argumento página fornecido ao man é normalmente o nome de um  programa,  utilitário  ou  função.  A  página  de  manual associada  com  esses  argumentos  é,  então,  localizada  e exibida. Uma seção, se fornecida, direcionará man para procurar apenas naquela seção  do  manual.  A  ação padrão  é  para  pesquisar em todas as seções disponíveis seguindo a seguinte ordem pré-definida (veja DEFAULTS), e para mostrar apenas a primeira  página  localizada, mesmo se página existir em várias seções.

---

-> Separado por sessões, man(1) `man 1 signal`

-> não só para comandos, para arquivos também 
`man fstab`


`apropos "system date"` procurou na descrição do comando

`whatis date` procurou na descrição do comando

`man -k` mesma coisa que o apropos

[103.1] Considerando a declaração:

`$CERT=LPIC-1`

Quais comandos exibiriam a saída abaixo?

`Este e um curso para LPIC-1 `

(Selecione 3 Respostas Corretas)

- echo Este e um curso para $CERT
- echo Este e um curso para "$CERT"
- echo 'Este e um curso para $CERT' X (não converte o $CERT)
- echo 'Este e um curso para \\$CERT' X (não converte o $CERT)
- echo Este e um curso para ${$CERT}
