*Áreas de Conhecimento:*
- Saber a prioridade padrão de um processo que é criado.
- Executar um programa com maior ou menor prioridade do que o padrão.
- Mudar a prioridade de um processo em execução.

`nice`
`ps`
`renice`
`top`

---

> MODIFICANDO AS PRIORIDADES DE EXECUÇÃO DOS PROCESSOS

`top` e `$ps -l` exibem informações e o valores de referência são `$PR` e `$NI` 
- `$PR` número interno que define a prioridade do processo (process)
- `$NI` propriedade que permite o usuário ou administror influenciar a prioridade (nice)

-> `Apenas o usuário root pode aumentar a prioridades` dos processos.
- `Os processos começam com nice zero por padrão.`
- `Se nada for definido no comando nice, define 10 por padrão`
- `Apenas o root pode definir um valor negativo`
- `Todos os usuários podem definir um valor positivo`

-> O `$NI` é alterado por dois programas:
- `nice` para definir o nice de um comando que não está em execução
- `renice` para alterar o nice de um processo que já está em execução

-> Os valores do `nice` podem ser definidos de [-20 até +19]
- `Valores negativos = mais prioridade` [‎-15]
- `Valores positivos = menos prioridade` [-15]
- `$nice -n 15` `$nice -15` `$nice --adjustment=15`

-> `renice` altera a prioridade de um processo que já está em execução
- `$renice -n 8 PID` `$renice 8 PID` `$renice --adjustment=8 PID`

- `Diferença entre nice e renice para indicação de valores negativos`
- `nice -15 (positivo) renice -15 (negativo)` [!!!]

-> Aplicando as mudanças de renice para um mesmo usuário ou grupo
- `$renice -n 8 -u user PID` `$renice -n 8 -g grupo PID`

 renice [-n] <‎prioridade> [-p|--pid] <‎pid>...

 renice [-n] <‎prioridade>  -u|--user <‎usuário>...

Altera a prioridade de processos em execução.

Opções:
 -n, --priority <‎núm> especifica o valor do "nice"
 -p, --pid            interpreta argumentos como ID de processo (padrão)
 -g, --pgrp           interpreta argumentos como ID de grupo de processo
 -u, --user           interpreta argumentos como nome ou ID de usuário
 -h, --help           exibe esta ajuda e sai
 -V, --version        exibe a versão

Para mais detalhes, veja renice(1).
