# 103.4

## Áreas de Conhecimento
- -> Redirecionamento da entrada padrão, da saída padrão e dos erros padrão.
- -> Canalização da saída de um comando à entrada de outro comando.
- -> Usar a saída de um comando como argumento para outro comando.
- -> Enviar a saída de um comando simultaneamente para saída padrão e para um arquivo.

`#tee`
`#xargs`

---

# Descritores
- 
Todo programa no linux recebe três descritores:

- Entrada padrão (STDIN) -> Saída Padrão (STDOUT)
- └──────────────────────-> Erro Padrão (STDERR)

A entrada padrão é o teclado, pode ser arquivo ou dispositivo.
A saída padrão/erro é a tela.


# REDIRECIONAMENTO DA SAÍDA PADRÃO

- -> `>` faz o resultado do comando ser enviado para um arquivo ao invés da tela, cria se não existir e sobreescreve se existir.
- -> `>‎>` faz o resultado do comando ser enviado para um arquivo, cria se não existir e faz um apendice na última linha se existir.
- -> `1>` e `1>‎>` são identicos ao `1>` e `1>‎>` pois o 1 é omitido. Significa o código do STDOUT.

# REDIRECIONAMENTO DO ERRO

- -> `2>` faz o resultado de erro do comando ser enviado para um arquivo ao invés da tela, cria se não existir e sobreescreve se existir.
- -> `2>‎>` faz o resultado do erro ser enviado para um arquivo, cria se não existir e faz um apendice na última linha se existir.
- -> `2>` e `2>‎>` significam o código do STDERR.
