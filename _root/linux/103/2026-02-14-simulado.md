<div class="quiz-q" data-answer="1" data-topic="103.4" markdown="1">
Qual dos comandos abaixo, redireciona a saída padrão do comando ls -l /etc/config* para o arquivo saida-ls.out e a saída de erro para o arquivo erros-ls.out?

1. ls -l /etc/config* >> saida-ls.out 2> erros-ls.out
Correto
2. ls -l /etc/config* < saida-ls.out 2> erros-ls.out
3. ls -l /etc/config* > saida-ls.out 2>&1 erros-ls.out
4. ls -l /etc/config* > erro-ls.out 2> saida-ls.out
5. ls -l /etc/config* >> saida-ls.out >> erro-ls.out 

<div class="quiz-explanation" markdown="1">
Os redirecionadores > e >> redireciona a saída padrão do comando, sendo que o >> indica que o conteúdo será adicionado ao final do arquivo indicado. Já o 2>, assim como o 2>>, redireciona a saída de erro da execução do comando. #bash

<div class="quiz-q" data-answer="4" data-topic="103.7" markdown="1">
Qual das afirmações abaixo é a mais correta sobre o resultado do comando:

grep -E "^[0-9][A-Z]" arquivo.out


1. Serão retornadas todas as linhas do arquivo.out que tenham um número seguido de uma letra qualquer
2. Serão retornadas todas as linhas do arquivo.out, exceto aquelas que contenham um número seguido de uma letra maiúscula
3. Serão retornadas todas as linhas que contenham letras ou números, excluindo-se as linhas em branco
4. Serão retornadas todas as linhas que tenham no começo da linha um número seguido por uma letra maiúscula
5. Serão retornadas todas as linhas que contenham no final da linha um número seguido por uma letra maiúscula 

<div class="quiz-explanation" markdown="1">
Ao se utilizar a opção -E no grep, a expressão será interpretada como uma expressão regular estendida. Na expressão, o símbolo ^ indica que a string deve estar no início da linha, seguido então de um número de 0 a 9, e uma letra maiúscula de A a Z. #grep
